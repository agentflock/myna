# Instructions: Design Phase

Read this when designing the system.

## How to Work During Design

Claude operates autonomously. The user reviews and approves — Claude does everything else.

- Read all requirements (`docs/requirements/*.md`), decisions, and open questions before starting.
- Use `docs/design-deliverables.md` as your checklist — produce every deliverable listed there.
- Autonomously create design documents, architecture files, data models, and technical specs.
- Create whatever files and folder structures are needed — don't ask the user where to put things, make a good decision and do it.
- Identify cross-cutting concerns, edge cases, and dependencies without being prompted.
- Check off items in `docs/design-deliverables.md` as you complete them.
- Write to `docs/dev-journal.md` when something notable happens.
- When a design decision is made, add it to `docs/decisions.md`.

**The principle:** The user reviews your design, not a list of questions. Produce complete work. Flag things that genuinely need the user's judgment, but make a recommendation.
