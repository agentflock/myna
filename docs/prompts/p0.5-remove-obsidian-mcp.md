# P0.5: Replace Obsidian MCP with Vault Operations Steering Skill

## Setup

**Model:** Sonnet | **Effort:** High

## Context

Myna currently ships an MCP server (`agents/mcp/myna-obsidian/`) that wraps Obsidian CLI for vault operations. Analysis shows that nearly all MCP operations can be done with Claude Code's built-in tools (Read, Write, Edit, Grep, Glob). The only genuinely irreplaceable feature is `eval` (Dataview queries), which is optional and rarely used.

Replacing the MCP with a steering skill that teaches Claude vault file patterns makes Myna a **pure markdown system** — no server processes, no Node.js build step, no runtime dependencies beyond Claude Code itself.

**Read these files:**
- `docs/architecture.md` — §6 (MCP Integration), §11 (Install output table)
- `docs/design/foundations.md` — §7 (MCP Tool Surface)
- The 24-skill table and 5 steering skills table from `docs/prompts/skill-rebuild-overview.md`

## What to do

### 1. Add a 6th steering skill to the architecture

Add `myna-steering-vault-ops` to the steering skills list. This skill teaches Claude how to interact with the vault using built-in tools instead of MCP calls.

**Content scope for `myna-steering-vault-ops`:**

**File I/O — use Claude Code built-in tools:**
- Read vault files → `Read` tool
- Write/create vault files → `Write` tool
- Append to or modify vault files → `Edit` tool
- Search file contents → `Grep` tool
- Find files by pattern → `Glob` tool

**Task queries — use Grep patterns instead of MCP `tasks` tool:**
- Open tasks: `Grep` for `- \[ \]` in vault files
- Completed tasks: `Grep` for `- \[x\]`
- Filter by project: match `[project:: {name}]` in the task line
- Filter by type: match `[type:: {type}]` (task, delegation, dependency, reply-needed, retry)
- Filter by person: match `[person:: {name}]`
- Filter by due date: match `📅 {date}` — compare against today for overdue detection
- Filter by status: match `[review-status:: pending]`
- Recurrence: match `🔁 every {interval}`
- Priority: match `⏫` (high), `🔼` (medium), `🔽` (low)

**Frontmatter operations — use Read + Edit:**
- Read frontmatter: `Read` the file, parse YAML between `---` markers
- Set frontmatter property: `Edit` the specific property line

**Backlink queries — use Grep:**
- Find files linking to `[[filename]]`: `Grep` for `\[\[filename\]\]` or `\[\[filename\|`

**Tag queries — use Grep:**
- Find files with a tag: `Grep` for `#tagname` (word boundary aware)
- List all tags: `Grep` for `#[a-zA-Z][\w-]*` across vault, unique

**Template creation — use Read + Write:**
- Read template from `_system/templates/{type}.md`
- Substitute variables: `{{name}}`, `{{date}}`, `{{project}}`, etc.
- Write the new file with substituted content
- If template doesn't exist, create a minimal file with frontmatter and appropriate tag

**Daily/weekly note paths:**
- Daily note: `Journal/DailyNote-{YYYY-MM-DD}.md`
- Weekly note: `Journal/WeeklyNote-{YYYY-MM-DD}.md` (Monday date)
- Contributions: `Journal/contributions-{YYYY-MM-DD}.md` (Monday date)
- Archive: `Journal/Archive/`

**File move (for journal archiving):**
- Use `Bash` with `mv` command to move files within the vault

**Search:**
- Full-text search: `Grep` with pattern across vault directory
- File name search: `Glob` with pattern

**What is NOT replaced (stays as external MCPs):**
- Email MCP — external service, user-provided
- Slack MCP — external service, user-provided
- Calendar MCP — external service, user-provided

These remain registered via `claude mcp add` and called by skills directly.
